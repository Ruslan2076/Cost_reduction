Sourcing Strategy: матрица Buy vs Build

|---|---|---|
| Кассовые операции | **SaaS (Buy)** — Cloud POS | Стандартная функция, конкурентного преимущества не даёт. Целевое состояние кейса — Cloud POS (тонкий клиент), локальные серверы ликвидируются. Подход миграции — Repurchase (Drop and Shop). Обязательное требование к выбору SaaS: offline-режим пробития чеков и работы с лояльностью (INC-8402) |
| Приём платежей (эквайринг) | **SaaS (Buy)** — Payment Gateway | Целевое состояние кейса: автоматическая адаптация под любые платёжные системы и терминалы через Payment Gateway. Устраняет разработку драйверов под каждый протокол (INC-8415) и жёсткую связку с кассой (Cost Driver 3) |
| Программа лояльности | **SaaS (Buy)** | Commodity. Консолидация локальных «тетрадок» и ERP-лояльности в единый облачный сервис, интегрированный с Cloud POS и Promo Engine |
| Расчёт премий, кадровый учёт | **SaaS (Buy)** | Стандартные HR-процессы, есть у всех — писать своё нецелесообразно. Устраняет локальный расчёт премий в магазинах |
| Печать ценников | **SaaS (Buy)** | Стандартная функция в составе Cloud POS / сопутствующих сервисов |
| Инкассация | **SaaS (Buy)** / аутсорс-сервис банка | Не ИТ-система, а стандартный процесс с банком-партнёром; собственная автоматизация не нужна |
| SKU Matching (миграция номенклатуры при M&A) | **PaaS (Build in Cloud)** | Уникальный процесс, критичный для бизнес-модели роста через M&A: сократить интеграцию точки с 2 недель до 5 дней. Готовых «коробок» под нечёткий маппинг разнородных баз с контролем человеком нет — SQL-скрипты fuzzy matching дают ложные срабатывания и риск продаж в минус (Артефакт 4). Собственный сервис на управляемых облачных компонентах (managed DB, очереди, ML-сервисы) |
| Локальные БД магазинов | **PaaS (Migrate)** → ликвидация | Replatform: замена MS SQL на сервере в подсобке на управляемую облачную БД. Закрывает Cost Driver 2 и класс инцидентов INC-8433; финально локальные серверы ликвидируются |
| Promo Engine | **PaaS/IaaS (Migrate)**, разработка in-house | Differentiator: единый движок промоакций — часть уникального ценообразования сети. Ядро не передаётся вендору (риск повторения сценария Lidl с перекройкой чужой «коробки»); хостинг переносится в облако |
| Smart Fulfillment (Click & Collect) | **PaaS/IaaS (Migrate)**, разработка in-house | Главное конкурентное преимущество сети — логистическая доступность. Покупка готового решения означала бы потерю контроля над ядром бизнеса; облако даёт масштабирование под постоянный приток новых точек |
| SAP ERP | **In-house/On-Premise (Tolerate)** | Мощная работающая система учёта; вне границ проекта снижения TCO магазинов. Замена или перенос создают риск, несопоставимый с целями проекта; модификация ядра под свои процессы — известный антипаттерн (кейс Lidl) |
| Кассовый узел на точке (планшет + фискальный регистратор) | **On-Premise (edge)** | Физическое оборудование в магазине не переносится в облако по определению; фискальный регистратор обязателен по требованиям законодательства о ККТ (вне источников — 54-ФЗ). Целевое состояние кейса: в магазине остаётся только планшет и фискальный регистратор |
