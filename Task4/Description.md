# Вывод о ROI миграции VitaStore (TCO, 3 года, 100 магазинов)

## Итоговые цифры

| Показатель | Значение |
|---|---|
| TCO AS-IS за 3 года | 68,2 млн ₽ |
| TCO TO-BE за 3 года | 52,9 млн ₽ |
| Экономия за 3 года | 15,3 млн ₽ (22,4%) |
| Снижение годовых затрат с года 2 (run-rate) | 32,1% (22,7 → 15,4 млн ₽/год) |
| Разовый CapEx TO-BE (edge-узлы + планшеты) | 7,0 млн ₽ |
| **Break-even point** | **~11–12 месяцев** |

## Резюме

Переход выгоден. Разовые вложения в оборудование целевой архитектуры (Local Edge Node + планшеты, 7,0 млн ₽) окупаются экономией операционных затрат примерно за 11–12 месяцев. Уже первый год TO-BE (22,07 млн ₽) дешевле года AS-IS (22,73 млн ₽) — CapEx перекрывается экономией на ФОТ и лицензиях. Со второго года сеть выходит на устойчивую экономию 7,3 млн ₽ ежегодно.

Главные источники экономии: ФОТ поддержки (−5,9 млн ₽/год: 7 выездных инженеров при локальных серверах против 2 специалистов облачной поддержки) и лицензии Windows Server / MS SQL, заменяемые подпиской на SaaS-кассу. Каналы связи в TO-BE дорожают (резервный LTE обязателен для offline-стратегии), но это перекрывается остальными статьями.

## Оговорки

1. **Цель бизнеса «TCO −40%» на горизонте модели достигается частично**: run-rate снижение — 32,1%. Полные 40% реалистичны при масштабировании со 100 до 500+ магазинов: облачный бэкенд и ФОТ поддержки в TO-BE почти не растут с числом точек, тогда как в AS-IS выездные инженеры и лицензии растут линейно.
2. Break-even рассчитан только по стоимости владения — затраты на внедрение и миграцию (разработка, проект перехода) исключены из scope по условию задачи; с их учётом срок окупаемости будет больше.
3. Все денежные значения — рыночные оценки РФ 2026 года (вне источников проекта: в файлах кейса цен нет). Модель построена на формулах: любое допущение меняется на листе «Допущения», итоги и break-even пересчитываются автоматически.
